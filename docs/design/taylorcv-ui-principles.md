# TaylorCV UI Principles

## What TaylorCV should feel like

TaylorCV should feel:

- Premium
- Trustworthy
- Sharp
- Calm
- Fast
- Useful
- Built for real job seekers
- More like a serious career product than a generic AI tool

TaylorCV should not feel:

- Like a cheap AI wrapper
- Like a resume template site
- Like a fake-growth landing page
- Like a generic Tailwind SaaS clone
- Like a dashboard full of decorative cards

## Design principles extracted from Claude Design prompt

Use the Claude design prompt as inspiration only. Do not copy it verbatim.

Useful extracted principles:

- Start from existing product context before designing.
- Good hi-fi design should not start from nothing.
- Inspect existing components, styles, spacing, typography, and visual vocabulary first.
- Ask clarifying questions only when ambiguity blocks quality.
- Prefer concrete design systems over vague “make it modern” instructions.
- Give multiple variations when exploring new visuals or flows.
- Use advanced CSS/layout thoughtfully.
- Avoid filler content.
- Avoid unnecessary iconography.
- Avoid AI-slop tropes.
- Verify the rendered result, not just the source code.
- Keep final summaries brief and practical.

## Visual language

Marketing pages:

- Deep navy, blue, teal, and controlled luminous accents.
- Dark premium backgrounds are allowed, but must not become noisy.
- White CV/product previews should be crisp and readable.
- Match score visuals should feel precise, not gimmicky.
- CTAs should be obvious and high contrast.

App workflow:

- Centered, focused, low-friction.
- The user should always know what step they are on.
- Avoid unnecessary screens.
- Prefer one clear primary action per stage.
- Loading states should feel intelligent and progress-based, not fake.

CV/document surfaces:

- White or near-white document preview.
- Strong text hierarchy.
- Blue accent for section headings/dividers only.
- Main body text remains dark.
- Keep layouts ATS-safe and one-page focused.

## Content rules

Every section must earn its place.

Do not invent:

- Fake testimonials
- Fake interview statistics
- Fake user counts
- Fake company logos
- Fake case studies
- Fake “trusted by” sections

Use believable early-stage proof:

- Built for New Zealand job seekers
- ATS-safe formatting
- One-page tailored CV
- Designed for students, graduates, tradespeople, and professionals
- Role-aware structure
- Evidence-backed bullet improvements

## Icon rules

Icons should be specific and useful.

Avoid:

- Repeating Sparkles, Wand, Bot, FileText, CheckCircle everywhere
- Decorative icon grids
- Icons that explain nothing
- Overly playful icons for serious workflow moments

Prefer:

- Minimal product-specific icons
- Custom small SVGs when needed
- Data/requirement/match/evidence metaphors
- Fewer icons with stronger purpose

## Motion rules

Motion should:

- Clarify progress
- Show cause and effect
- Make state changes feel smooth
- Support perceived speed
- Feel calm and premium

Avoid:

- Random floating cards
- Constant pulsing
- Excessive shimmer
- Over-bouncy transitions
- Animations that delay the task

## QA checklist

Before finishing important UI work, check:

- Does the page explain the product in 5 seconds?
- Is the main CTA visually dominant?
- Does the screen feel premium without being noisy?
- Is there any fake or exaggerated proof?
- Is spacing consistent?
- Is typography confident?
- Are mobile layouts usable?
- Are hover/focus/loading/error states handled?
- Is the UI specific to TaylorCV, or could it belong to any AI SaaS?
