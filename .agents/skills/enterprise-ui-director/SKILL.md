# Enterprise UI Director

Use this skill when the task involves UI, layout, visual polish, design systems, premium SaaS quality, or improving how TaylorCV feels.

## Goal

Make TaylorCV look and feel like a serious, premium, trustworthy career platform for the New Zealand market.

Do not produce generic AI SaaS UI.

## Before editing

1. Identify the route, component, and current visual pattern.
2. Inspect nearby components and global styles.
3. Look for existing spacing, typography, color, border, radius, and shadow patterns.
4. Reuse existing design vocabulary unless it is clearly weak.
5. If the task references a screenshot or design, translate it into reusable product rules rather than blindly copying pixels.

## Visual quality rules

Prioritize:

- Clear hierarchy
- Strong alignment
- Controlled whitespace
- Premium typography
- Crisp cards and panels
- Purposeful color
- Legible contrast
- Consistent border radii
- Subtle depth
- Product-specific visual metaphors

Avoid:

- Generic dashboard cards
- Repetitive icon rows
- Excessive gradients
- Fake glass effects
- Emoji
- Left-border cards
- Random decorative blobs
- Overused AI sparkles
- Weak gray-on-gray text
- Unclear CTA hierarchy

## TaylorCV-specific direction

For marketing:

- Use premium navy/blue/teal direction.
- Keep visuals conversion-focused.
- Use white CV previews as the main product anchor.
- Show role matching, requirements, evidence, gap questions, and tailored CV improvement clearly.

For app workflow:

- Keep screens focused and low-friction.
- Avoid unnecessary confirmation pages.
- Make progress feel intelligent.
- Keep the user moving toward the CV result.

For CV previews:

- Use blue accents only for section labels, dividers, links, and selected highlights.
- Keep body text dark.
- Preserve one-page, ATS-safe clarity.

## Output expectations

When implementing:

1. Make the UI change.
2. Add hover/focus/loading/empty/error states where relevant.
3. Check responsive behavior.
4. Run the screen.
5. Inspect the rendered result.
6. Polish spacing, hierarchy, alignment, and contrast.
7. Run lint/build/typecheck where appropriate.

Do not finish if the result looks average.
